1.       Default Set_Classification Problem

Before splitting the data set intro training and test sets, we made some analyses. Although student was a categorical variable, we encoded it as a 1-0 numeric variable.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

According to the density graph above, income and balance features span a wide range, and the difference between the levels of student is not very large. Therefore, none of them seems to be biased.

On the other hand, this problem is an unbalanced classification problem. The number of No is 9667 while the number of Yes is 333 in the response variable, which is default. This unbalance may cause problems. By keeping this in mind, we tried two methods. Firstly, we did not do anything about class unbalance. We paid attention to the split the training set whose distribution resembles the distribution of the main data set in terms of default response variable. Secondly, we used SMOTE function to deal with class unbalance problem. We used balanced training and test set. Since kNN considers distance matrix, normalization or standardization of the features is important. We chose standardization of the features.

In the first application, we evaluated the predictive performance of 10 kNN models with 10 different k values. Best error and specificity values are obtained when k = 10. However, sensitivity fluctuates throughout different k values. One can choose either k = 9 or k = 10 depending on their priorities. Here, we consider k = 10 as the best k value. The results are error = 0.03, sensitivity = 0.2934 and specificity = 0.996. Our findings about error, sensitivity and specificity are shown below.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

In the second application, before splitting the data set we use SMOTE function. SMOTE function is a widely suggested function for dealing with unbalanced classification problems. By using SMOTE, we reduce the number of instances with No label and increased the number of intances with Yes label. Therefore, the sizes of training and test set are smaller in this application so that the number of Yes and No under default response variable are almost the same. We performed the same method as we use in the first application. We evaluate 10 different k values.

 

However, the results get worse. None of the metrics reveals a stable trend. Moreover error rates increases to 0.115 on the average and specificity decreases as k increases. Therefore, please consider the first application.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

2.       Toyota Regression Problem

In this problem, we performed randomforest since on the second part, an analysis on its parameters is demanded.

From the correlation plot of features and Price shown below, we can observe the followings:

# Age seems to be highly correlated with Price, meaning that it can stand out as an important factor.

# Weight and KM seem to be correlated with Price at some degree too.

# However, CC and Doors are uncorrelated with Price.

# Apart from the correlation between features and Price, one should also consider the correlations between features. CC and Weight are correlated but it is not strong.

# As expected, Age and KM is correlated as well. However it is weaker than the correlation between CC and Weight.

By looking at the density graph of the features and the response variable (shown below), we can say that most of the cars are not automatic. Therefore, we do not expect that this feature will be significant.

In splitting the data set into training and test set, we tried to maintain the same histogram for both the whole data set and the training data set. We used rmse as an error measure. By configuring mtry = 1,2,3,4,5,6,7,8,9; ntree = 100,300,500,700,900; and nodesize = 1,3,5,7,10, in total we checked 9*5*5 = 225 random forest models. The best model in terms of error is obtained by mtry = 4, ntree = 500 and nodesize = 10. The rmse error of this model was roughly 1049.

RMSE = function(actual, predicted){

sqrt(mean((actual - predicted)^2))

}

rmse_all = data.table()

imp_all = data.table()

 

for(m in mtry_options){

for(n in ntree_options){

for(s in node_options){

assign(paste0("rf_",m,"_",n,"_",s),rf) # to call back for variable importance

rmse_error = RMSE(test_dt$Price,rf_prediction)

rmse_all = rbind(rmse_all,

data.table(mtry = m, ntree = n, nodesize = s, error = rmse_error))

# variable importance

imp = data.table(t(importance(rf, type = 1, scale = FALSE)))

imp_all = rbind(imp_all,

data.table(mtry = m, ntree = n, nodesize = s, imp))

}

}
